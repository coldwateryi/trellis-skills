# License 与安全策略

## License 分类

| License | 默认处理 |
| --- | --- |
| MIT / Apache-2.0 / BSD | 可吸收思路；搬运文本或脚本仍需注明来源并保持最小化 |
| GPL / AGPL / LGPL | 只做思路级总结，不搬运文本、脚本或模板到 MIT skill |
| 未声明 license | 只做公开事实摘要，不吸收可版权化文本或代码 |
| 商业/非商用限制 | 默认不吸收实现和模板 |
| 不可信来源 | 不吸收 |

## 允许吸收

- 抽象工作流。
- 非版权化的工程原则。
- 自己重新表述的流程结构。
- 用 Trellis 术语重写的 checklist 项。
- 由项目启发但独立设计的验证场景。

## 禁止吸收

- 大段复制 `SKILL.md` 或 references 原文。
- 直接搬运脚本。
- 保留第三方项目内部私有路径、组织规则或密钥说明。
- 绕开原 license 的模板重写。
- 未经确认引入强 copyleft 依赖。

## 输出要求

每份 absorption card 必须写明：

- license 判定。
- 哪些内容只做思路参考。
- 哪些内容可安全重写。
- 是否需要人工或法务确认。
