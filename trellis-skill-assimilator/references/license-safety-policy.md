# License and Safety Policy

## License Classes

| License | Default handling |
| --- | --- |
| MIT / Apache-2.0 / BSD | Ideas may be absorbed; moved text or scripts still require attribution and minimal copying |
| GPL / AGPL / LGPL | Ideas only; do not move text, scripts, or templates into MIT skills |
| No declared license | Public facts only; do not absorb copyrightable text or code |
| Commercial / non-commercial restriction | Do not absorb implementation or templates by default |
| Untrusted source | Do not absorb |

## Allowed to Absorb

- Abstract workflows.
- Non-copyrightable engineering principles.
- Independently reworded process structure.
- Checklist items rewritten in Trellis terms.
- Verification scenarios independently designed from the idea.

## Not Allowed

- Copying large passages from `SKILL.md` or references.
- Moving scripts directly.
- Preserving third-party private paths, organization rules, or secret-handling notes.
- Rewriting templates to bypass the source license.
- Introducing strong copyleft dependencies without confirmation.

## Output Requirements

Each absorption card must state:

- License judgment.
- Which content is idea-only.
- Which content can be safely rewritten.
- Whether human or legal review is needed.
