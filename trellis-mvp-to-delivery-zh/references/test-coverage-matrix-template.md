# 测试覆盖矩阵模板

规划或补齐完整交付测试时使用此模板。

## 测试覆盖矩阵

| 需求 ID | 测试类型 | 测试文件 | 测试用例 | 当前状态 | 缺口 | 人工验证原因 | 人工验证步骤 |
| --- | --- | --- | --- | --- | --- | --- | --- |
| REQ-001 | unit |  |  | MISSING |  |  |  |

允许的测试类型：

- `unit`
- `integration`
- `e2e`
- `smoke`
- `regression`
- `manual`

覆盖规则：

1. 每个 `REQ-*` 和 `AC-*` 必须映射到至少一个测试或人工验证条目。
2. 优先覆盖核心成功路径、失败路径、边界条件、历史回归风险和跨模块数据流。
3. 不要为了覆盖率数字编写无意义测试。
4. 如果不适合自动化，说明原因并写清楚人工验证步骤。
